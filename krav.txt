Beskrivning
Du ska skapa en webbsida där en användare kan söka på foton från Flickr. 
Se instruktioner: https://docs.google.com/document/d/1FWe3baEgyMKm7buw0Hz3xCDJpMiv79C3pwMYtVgkWZU/edit 

Krav
* Du får skriva HTML, CSS och JS precis som du vill så länge du håller dig till rule of seperation. Alltså, HTML-filer innehåller endast html, ingen css eller js!
* Sökformulär
Ett formulär där användaren kan ange
- Fri söktext
- Antal foton som ska visas
- Storlek på fotona - Tre olika alternativ: stor, mellan eller liten
- Vilket sätt fotona ska sorteras - Efter datum de laddades upp på flickr, efter relevans eller efter “interestingness” 

Tips: Vi har mest använt text och number inputs men kom ihåg att det finns flera andra, tex radio-input och select för drop down meny. 


Sökresultat
- Fotona i sökresultatet från Flickr ska visas på valfritt sätt på sidan men måste
- Hämtas med Flickrs API 
- Innehålla det som användaren sökt på
- Vara lika många som användaren angett
- Visas i den storleken användaren angett. Dvs storleken får ej ändras i css utan ska vara den storleken som hämtats från API:t. 
- Sorteras på det sättet användaren angett


Animation 
- Under tiden man väntar på svar från databasen ska det visas en animation. Alltså mellan det att användaren har sökt och att fotona visas på sidan. 
- Det brukar inte ta lång tid att få svar från databasen så animationen behöver inte vara lång.
- Du väljer själv vad för redskap du använder till animationen, css animation, jQuery, anime.js eller något annat. 


Bibliotek
- Du behöver använda minst ett bibliotek. Använder du ett för animationen är det här kravet uppfyllt. Bootstrap kommer inte räknas som ett js bibliotek. 


Error
- Alla error relaterat till sökningen ska hanteras med ett användarvänligt meddelande till användaren på webbsidan.
Tips: I den mån det går kan du se till att användaren inte kan orsaka ett error. Till exempel att text-inputen inte får vara tom. 



Betyg G
För att projektet ska bli godkänt behöver allt under krav finnas med och fungera.
Projektet betygsätts inte utifrån utseende. Men all info måste vara tydlig.
Projektet ska vara inlämnat i tid enligt anvisningarna under ‘Inlämning’.

Betyg VG
För att projektet ska få Väl Godkänt behöver alla kriterier under G uppfyllas. 
Projektet behöver ha någon extra funktionalitet utöver det som finns listat under krav. Fråga mig för säkerhets skull om det är tillräckligt för VG nivå.
Koden samt alla filer behöver vara strukturerat på ett sätt som gör det readable och maintainable. 

Tips!
Tänk igenom med papper och penna först.
Dela upp allting i sina minsta moment.
Fokusera på ETT moment i taget och se till att ett moment fungerar innan du går vidare till nästa.
Våga ta en paus och våga be om hjälp!
